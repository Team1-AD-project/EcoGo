\subsection{DAST (Dynamic Application Security Testing)}

\subsubsection{Tool 12: OWASP ZAP}

\textbf{Purpose}: Runtime vulnerability scanning of deployed application

\textbf{Configuration}:
\begin{itemize}
    \item \textbf{Files}: \texttt{.zap/zap-config.yaml}, \texttt{.zap/rules.tsv}
    \item \textbf{Scan Types}: Baseline + Full Scan
    \item \textbf{Max Duration}: 8 minutes (3 min spider + 5 min active scan)
\end{itemize}

\textbf{ZAP Configuration} (\texttt{.zap/zap-config.yaml}):
\begin{lstlisting}[language=bash]
env:
  contexts:
    - name: EcoGo API
      urls:
        - http://localhost:8090/api/v1/.*
      includePaths:
        - http://localhost:8090/api/v1/.*
      excludePaths:
        - http://localhost:8090/actuator/.*

jobs:
  - type: spider
    parameters:
      maxDuration: 3
      maxChildren: 10
  
  - type: passiveScan-wait
    parameters:
      maxDuration: 2
  
  - type: activeScan
    parameters:
      maxDuration: 5
      maxRuleDurationInMins: 1
\end{lstlisting}

\textbf{CI/CD Integration}:
\begin{lstlisting}[language=bash]
# Baseline Scan
docker run --network="host" -v $(pwd):/zap/wrk/:rw \
  -t zaproxy/zap-stable zap-baseline.py \
  -t http://localhost:8090 \
  -r testreport.html \
  -w testreport.md

# Full Scan with Custom Config
docker run --network="host" -v $(pwd):/zap/wrk/:rw \
  -t zaproxy/zap-stable zap-full-scan.py \
  -t http://localhost:8090 \
  -c .zap/zap-config.yaml \
  -r zap-report.html
\end{lstlisting}

\textbf{Test Coverage}:
\begin{itemize}
    \item SQL Injection
    \item Cross-Site Scripting (XSS)
    \item Security Misconfigurations
    \item Broken Authentication
    \item Sensitive Data Exposure
    \item Missing Security Headers
    \item CSRF vulnerabilities
    \item XML External Entities (XXE)
    \item Insecure Deserialization
\end{itemize}

\textbf{Scan Status}:
\begin{tcolorbox}[colback=yellow!10, colframe=orange!75!black, title=ZAP Scan Note]
In the current CI/CD run, the ZAP scan completed but did not generate detailed vulnerability reports. This is common in early pipeline runs where the application may not have been fully accessible during the DAST stage, or scan configuration requires tuning.

Future runs should verify:
\begin{itemize}
    \item Application is fully started before ZAP scan
    \item Network connectivity between ZAP container and application
    \item Scan timeout settings are appropriate
\end{itemize}
\end{tcolorbox}
